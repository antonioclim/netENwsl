A Signal Across the Atlantic

The telegraph that changed everything

Historical Context

On 16 August 1858, Queen Victoria sent the first transatlantic telegraph message to US President James Buchanan. The 98-word message took 16 hours to transmit‚Äîa journey that previously required weeks by ship.

Within three weeks, the cable failed due to signal degradation and voltage overloads. It would take eight more years to establish a reliable connection.

The challenges faced then‚Äîsignal attenuation, noise, synchronisation‚Äîare the same problems we solve today at the Physical Layer.

üéØ Why This Matters

Every WhatsApp message, every database query, every API call must eventually become electrical signals on a wire or radio waves in the air.

Understanding Layers 1 and 2 helps you diagnose why packets "disappear", why latency spikes occur, and why some networks are faster than others.

01

The Physical Layer

Transforming bits into signals‚Äîthe interface between digital logic and the analogue world

What Is the Physical Layer?

Layer 1 in the OSI model

Formal Definition

The Physical Layer is responsible for the transmission and reception of raw, unstructured bit streams over a physical medium. It defines the electrical, optical, or radio specifications for activating, maintaining, and deactivating the physical link between end systems.

üì° What It Defines

  • Signal type: Electrical, optical, electromagnetic
  • Voltage levels: What represents 0 and 1
  • Bit rate: Speed in bps, Mbps, Gbps
  • Synchronisation: Timing between sender/receiver
  • Physical connectors: RJ-45, LC, SC, antennas

‚ö†Ô∏è What It Cannot Do

  • Address devices (no MAC, no IP)
  • Detect or correct errors
  • Understand message boundaries
  • Communicate directly with software

These limitations are why Layer 2 exists.

The Postal Analogy

Understanding layers through a familiar system

Analogy

Think of sending a parcel by post. The Physical Layer is the road, rail, and air infrastructure‚Äîit moves packages from one location to another without understanding what is inside. It does not know if the package contains a birthday card or legal documents; it simply transports physical matter.

üõ£Ô∏è Physical Layer

Roads, trucks, aeroplanes‚Äîthe transport infrastructure

Moves bits without understanding them

üì¶ Data Link Layer

Packaging, labels, tracking numbers‚Äîthe delivery system

Organises bits into frames with addresses

üìù Network Layer+

Addresses, routing decisions‚Äîthe sorting centres

Determines the path across networks

Key Insight

Just as road quality affects delivery speed (potholes = delays), physical media quality affects network performance (noise = errors, distance = attenuation).

Twisted Pair Cable

The most ubiquitous network medium

üîß How It Works

Two insulated copper wires are twisted together to reduce electromagnetic interference (EMI). The twist causes external noise to affect both wires equally, which can then be cancelled out by the receiver (differential signalling).

UTP

Unshielded Twisted Pair

Most common. Cat5e/Cat6/Cat6a. Cost-effective but susceptible to interference near power cables.

STP

Shielded Twisted Pair

Additional foil/braid shielding. Used in industrial environments or where EMI is high.

Cat Ratings

Category performance

Cat5e: 1 Gbps @ 100m
Cat6: 10 Gbps @ 55m
Cat6a: 10 Gbps @ 100m

Fibre Optic Cable

Light-speed data transmission

Principle of Operation

Fibre optic cables transmit data as pulses of light through a glass or plastic core. The light is kept within the core by total internal reflection‚Äîthe boundary between the core and cladding acts as a perfect mirror when light hits it at a shallow angle.

Single-Mode Fibre (SMF)

  • Core diameter: ~9 ¬µm
  • One light path (mode)
  • Distance: up to 80+ km
  • Higher cost, requires laser
  • Use: WAN, backbone, undersea

Multi-Mode Fibre (MMF)

  • Core diameter: 50‚Äì62.5 ¬µm
  • Multiple light paths
  • Distance: up to 550 m
  • Lower cost, uses LED
  • Use: LAN, data centres

Common Misconception

"Fibre is always faster than copper." Not quite‚Äîboth can achieve 10+ Gbps. Fibre's advantage is distance (no signal degradation) and immunity to EMI. For short runs in an office, Cat6a copper may be more practical.

Signal Degradation

Why signals weaken and distort

üìâ Attenuation

Signal weakens with distance. Energy is lost to resistance (copper) or absorption (fibre).

Solution: Repeaters, amplifiers, shorter cable runs

‚ö° Noise

External interference corrupts signals. Sources: motors, fluorescent lights, adjacent cables.

Solution: Shielding, differential signalling, error detection

üåÄ Distortion

Signal shape changes during transmission. Dispersion in fibre, reflections in copper.

Solution: Equalisation, better encoding

üî¨ Crosstalk in Twisted Pair

Crosstalk occurs when signals in one wire pair induce interference in adjacent pairs within the same cable. This is particularly problematic at higher frequencies.

  • NEXT (Near-End Crosstalk): Interference measured at the transmitting end
  • FEXT (Far-End Crosstalk): Interference measured at the receiving end

Cat6a cables use tighter twists and better separation to minimise crosstalk at 10 Gbps.

Line Coding

The Problem

Line coding defines how binary data (0s and 1s) is represented as voltage levels on a wire. The challenge: the receiver must stay synchronised with the sender. Long sequences of identical bits (e.g., 00000000) provide no transitions for the receiver's clock to lock onto.

Simplest encoding:

Modern Encoding Schemes

Balancing efficiency and synchronisation

4B/5B

Used in Fast Ethernet (100BASE-TX)

  • 4 data bits ‚Üí 5 transmitted bits
  • 25% overhead (vs 100% for Manchester)
  • Codes chosen to limit consecutive identical bits

8B/10B

Used in Gigabit Ethernet (1000BASE-T)

  • 8 data bits ‚Üí 10 transmitted bits
  • 25% overhead
  • DC balance maintained

  • 4 voltage levels per symbol
  • 2 bits per symbol
  • Doubles throughput at same baud rate

Critical Insight

The evolution from Manchester (2 symbols/bit) to 4B/5B (1.25 symbols/bit) to PAM-4 (0.5 symbols/bit) shows how encoding efficiency has improved. This matters because higher baud rates require more expensive hardware‚Äîefficient encoding lets us achieve higher data rates with existing infrastructure.

Modulation

Encoding data for wireless transmission

Why Modulation?

Wireless signals require a carrier wave‚Äîa high-frequency signal that can propagate through air. Modulation varies a property of the carrier (amplitude, frequency, or phase) to encode information. The receiver demodulates to extract the original data.

ASK

Amplitude Shift Keying

Varies signal strength

  • 1 = High amplitude
  • 0 = Low/no amplitude

Simple but noise-sensitive

FSK

Frequency Shift Keying

Varies signal frequency

  • 1 = Higher frequency
  • 0 = Lower frequency

Used in older modems

PSK

Phase Shift Keying

Varies signal phase

  • 0¬∞ = 00, 90¬∞ = 01
  • 180¬∞ = 10, 270¬∞ = 11

More bits per symbol

QAM: The Best of Both

Quadrature Amplitude Modulation combines amplitude and phase variation. Modern WiFi uses 256-QAM or 1024-QAM, encoding 8‚Äì10 bits per symbol!

The Data Link Layer

From raw bits to structured frames‚Äîadding addressing, framing, and error detection

What Is the Data Link Layer?

Layer 2 in the OSI model

Formal Definition

The Data Link Layer provides node-to-node data transfer‚Äîa link between two directly connected nodes. It handles the physical addressing of devices (MAC addresses), organises bits into frames, provides error detection, and manages access to the shared physical medium.

üìã Core Responsibilities

  • Framing: Delimiting data units
  • Addressing: MAC addresses (48-bit)
  • Error detection: CRC/FCS
  • Media access control: Who transmits when
  • Flow control: Preventing receiver overflow

üèóÔ∏è Two Sublayers

LLC (Logical Link Control)
IEEE 802.2 ‚Äî Interface with Network Layer, protocol multiplexing

MAC (Media Access Control)
IEEE 802.3/11 ‚Äî Interface with Physical Layer, addressing, access control

Imagine receiving a continuous stream of letters: HELLOWORLDHOWAREYOU. Without spaces or punctuation, it's difficult to parse. Is it "HELLO WORLD HOW ARE YOU" or "HELL OWO RLD..."? The Physical Layer delivers a continuous bit stream‚ÄîLayer 2 must add the "punctuation" through framing.

MAC Addresses

Physical addressing at Layer 2

Media Access Control Address

A MAC address is a 48-bit (6-byte) hardware identifier assigned to network interface cards. It is designed to be globally unique and permanent (though software can override it). MAC addresses operate only within a local network segment‚Äîthey do not cross routers.

Error Detection: CRC

Cyclic Redundancy Check

How CRC Works

A Cyclic Redundancy Check treats the data as a large binary number and divides it by a fixed polynomial. The remainder of this division is appended to the frame as the FCS. The receiver performs the same division‚Äîif the remainder is non-zero, the frame is corrupted.

üìä CRC Properties

  • Detects all single-bit errors
  • Detects all double-bit errors
  • Detects all odd numbers of errors
  • Detects burst errors shorter than CRC width
  • CRC-32: Used in Ethernet, detects >99.9999% of errors

üêç Python Implementation

import zlib

def crc32(data: bytes) -> int:
    """Calculate CRC32 checksum."""
    return zlib.crc32(data) & 0xFFFFFFFF

# Usage from proto_common.py
checksum = crc32(header + payload)

This is exactly what the starterkit's binary protocol uses!

Detection vs Correction

CRC can detect errors but not correct them. If a frame fails CRC, it is silently discarded. Upper layers (TCP) handle retransmission. Some systems use Forward Error Correction (FEC) to correct errors without retransmission‚Äîcommon in satellite links where latency is high.

CSMA/CD

Carrier Sense Multiple Access with Collision Detection

The Shared Medium Problem

When multiple devices share a single transmission medium (e.g., coaxial cable, hub), they need a protocol to decide who transmits when. CSMA/CD was the original Ethernet solution: listen before transmitting, and if two stations transmit simultaneously, detect the collision and retry.

üîÑ CSMA/CD Algorithm

Historical Context

CSMA/CD is obsolete in modern networks. Full-duplex switched Ethernet eliminated collisions entirely‚Äîeach port has a dedicated collision domain of one. However, understanding CSMA/CD helps you grasp why minimum frame sizes exist (64 bytes = collision detection window).

CSMA/CA

Carrier Sense Multiple Access with Collision Avoidance

The Hidden Node Problem

In wireless, you cannot detect collisions while transmitting‚Äîyour own signal drowns out everything else. Additionally, two stations might both be in range of an access point but not each other (hidden nodes). CSMA/CA avoids collisions rather than detecting them.

üîÑ CSMA/CA Algorithm

  1. Listen to the medium
  2. If busy, wait until idle
  3. Wait DIFS (DCF Inter-Frame Space)
  4. Wait random backoff time
  5. Transmit frame
  6. Wait for ACK from receiver
  7. If no ACK, double backoff and retry

‚úã RTS/CTS (Optional)

Request to Send / Clear to Send

For large frames, sender first transmits a small RTS frame. If the medium is clear, receiver responds with CTS. This "reserves" the channel and mitigates hidden node problems.

Adds overhead‚Äîtypically disabled for small frames.

WiFi vs Ethernet Latency

CSMA/CA's mandatory wait times and ACKs make WiFi latency more variable than Ethernet. For real-time applications (gaming, VoIP), wired connections remain superior.

Switches & CAM Learning

How switches learn device locations

The Switch's Job

A switch is a Layer 2 device that forwards frames based on MAC addresses. Unlike a hub (which broadcasts to all ports), a switch learns which MAC addresses are reachable through which ports and forwards frames only to the appropriate port.

üìö Learning Process

  1. Frame arrives on port X
  2. Read source MAC
  3. Store: MAC ‚Üí Port X in CAM table
  4. Look up destination MAC
  5. If found: forward to that port
  6. If not found: flood to all ports (except source)

‚è±Ô∏è CAM Aging

Entries expire after a timeout (typically 300 seconds). This handles:

  • Devices moving to different ports
  • Devices leaving the network
  • Memory management

Virtual LANs (VLANs)

Logical network segmentation

The Broadcast Problem

In a flat Layer 2 network, broadcasts reach every device. As the network grows, broadcast traffic consumes bandwidth and CPU. VLANs create separate broadcast domains on a single physical switch, improving security and performance.

ARP: Bridging L2 and L3

Address Resolution Protocol

The Problem

To send an IP packet, the sender needs the destination's MAC address. But applications only know IP addresses. ARP resolves IP addresses to MAC addresses within a local network segment.

üîÑ ARP Request/Reply

Security: ARP Spoofing

ARP has no authentication. An attacker can send fake ARP replies, associating their MAC with another IP (man-in-the-middle attack). Mitigations include Dynamic ARP Inspection (DAI), static ARP entries, and 802.1X.

Common Misconceptions

What students often get wrong

‚ùå Misconception

"MAC addresses are used to route packets across the Internet."

‚úì Reality

MAC addresses are local only. They change at every router hop. IP addresses provide end-to-end addressing; MAC addresses provide hop-by-hop delivery.

‚ùå Misconception

"Switches and hubs are the same thing."

‚úì Reality

Hubs broadcast to all ports; switches learn MAC addresses and forward only to the correct port. Switches create separate collision domains per port.

‚ùå Misconception

"Fibre is always faster than copper."

‚úì Reality

Both can achieve 10+ Gbps. Fibre's advantage is distance (low attenuation) and EMI immunity, not raw speed per se.

Historical Context: BSD Sockets

The C heritage of network programming

The Origin

The socket API was developed at UC Berkeley in the early 1980s for BSD Unix. Python's socket module is essentially a wrapper around the same C system calls. Understanding the C context helps you appreciate why the Python API looks the way it does.

/* C: Create and bind a socket */
int sockfd = socket(AF_INET, SOCK_STREAM, 0);

struct sockaddr_in addr;
addr.sin_family = AF_INET;
addr.sin_port = htons(5400);  // Host to network short
addr.sin_addr.s_addr = INADDR_ANY;

bind(sockfd, (struct sockaddr*)&addr, sizeof(addr));
listen(sockfd, 16);
# Python: Same operations
import socket

sockfd = socket.socket(socket.AF_INET, socket.SOCK_STREAM)

addr = ('0.0.0.0', 5400)
sockfd.bind(addr)
sockfd.listen(16)

# htons() is automatic in Python!

Why Network Byte Order Matters

Different CPUs store multi-byte integers differently (little-endian vs big-endian). Network protocols use big-endian (network byte order). The ! in Python's struct.pack("!H", ...) means "use network byte order"‚Äîequivalent to C's htons().
