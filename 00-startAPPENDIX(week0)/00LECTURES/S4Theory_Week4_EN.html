Detectie Erori: CRC

Cyclic Redundancy Check

Cum Works CRC

A Verificare Redundant Ciclica treats the data as a large binary number and divides it by a fixed polynomial. Interfata remainder of this division is appended to the frame as the FCS. Interfata receiver performs the same division‚Äîif the remainder is non-zero, the frame is corrupted.

üìä CRC Properties

  • Detecteaz all errorsle of un singur bit
  • Detects all double-bit errors
  • Detects all odd numbers of errors
  • Detects burst errors shorter than CRC width
  • CRC-32: Used in Ethernet, detects >99.9999% of errors

Detection vs Correction

CRC can detect errors but not correct them. If a frame fails CRC, it is sislowly discarded. Upper layers (TCP) handle retransmission. Some systems use Forward Error Correction (FEC) to correct errors without retransmission‚Äîcommon in satellite links where latency is high.

CSMA/CD

Carrier Sense Multiple Access with Collision Detection

Interfata Shared Medie Problem

When multiple devices share a single transmission medium (e.g., coaxial cable, hub), they need a protocol to decide who transmits when. CSMA/CD was the original Ethernet solution: listen befhours transmitting, and if two stations transmit simultaneously, detect the collision and retry.

üîÑ CSMA/CD Algorithm

Context Istoric

CSMA/CD is obsolete in modern networks. Full-duplex switched Ethernet eliminated collisions entirely‚Äîeach port has a dedicated collision domain of one. However, understanding CSMA/CD helps you grasp why minimum frame sizes exist (64 bytes = collision detection window).

CSMA/CA

Carrier Sense Multiple Access with Collision Avoidance

Interfata Hidden Node Problem

In wireless, you cannot detect collisions while transmitting‚Äîyour own signal drowns out everything else. Additionally, two stations might both be in range of an access point but not each other (hidden nodees). CSMA/CA avoids collisions rather than detecting them.

üîÑ CSMA/CA Algorithm

  1. Listen to the medium
  2. If ocupat, wait until idle
  3. Wait DIFS (DCF Inter-Frame Space)
  4. Wait random backoff time
  5. Transmite framel
  6. Wait for ACK from receiver
  7. If no ACK, double backoff and retry

‚úã RTS/CTS (Optional)

Request to Send / Clear to Send

For large frames, sender first transmits a small RTS frame. If the medium is clear, receiver responds with CTS. This "reserves" the channel and mitigates hidden nodee problems.

Adds overhead‚Äîtypically disabled for small frames.

WiFi vs Ethernet Latency

CSMA/CA's mandatory wait times and ACKs make WiFi latency mhours variable than Ethernet. For real-time applications (gaming, VoIP), wired connections remain superior.

Switch-uri and Invatare CAM

How switches learn device locations

Interfata Switch's Job

A switch is a Nivel 2 device that forwards frames based on MAC addresses. Alike a hub (which broadcasts to all ports), a switch learns which MAC addresses are reachable through which ports and forwards frames only to the appropriate port.

ARP: Punte between L2 and L3

Adres Resolution Protocol

Problema

To send an IP packet, the sender needs the destination's MAC address. But applications only know IP addresses. ARP resolves IP addresses to MAC addresses within a local network segment.

Conceptii Gresite Comune

What students often get wrong

‚ùå Misconception

"MAC addresses are used to route packets across the Internet."

‚úì Realitate

MAC addresses are local only. Interfatay change at every router hop. IP addresses provide end-to-end addressing; MAC addresses provide hop-by-hop delivery.

‚ùå Misconception

"Switches and hubs are the same thing."

‚úì Realitate

Hubs broadcast to all ports; switches learn MAC addresses and forward only to the correct port. Switches create separate collision domains per port.

‚ùå Misconception

"Fibre is always faster than copper."

‚úì Realitate

Both can achieve 10+ Gbps. Fibre's advantage is distance (low attenuation) and EMI immunity, not raw speed per se.
