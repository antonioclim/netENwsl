üñ•Ô∏è Working Environment: WSL2 + Docker

The labs use a modern stack for network simulation:

ü™ü WSL2 (Windows Subsystem for Linux)

What it is: A real Linux kernel running in Windows

Why: Networking tools (Wireshark, tcpdump) work natively

Access: Open "Ubuntu" from Start Menu or type wsl in PowerShell

üê≥ Docker

What it is: Isolated containers for simulating network servers and services

Why: You can run DNS, HTTP, FTP servers without installing them on the system

Start: sudo service docker start

üéõÔ∏è Portainer

What it is: Web interface for managing Docker containers

Why: Easily visualise running containers and their logs

Access: http://localhost:9000

üêç Python 3

What it is: The language in which exercises are written

Why: Clear syntax, powerful networking libraries

Check: python3 --version

verification_mediu.sh 
# Verify all components work
python3 --version          # Python 3.10+
docker --version           # Docker 20+
docker ps                  # You should see Portainer

What is the main purpose of using Docker in networking labs?
A Faster Python code execution
B Editing Python files
C Simulating network servers and services in isolated containers
D Automatic Python installation
