üìù Basic Usage

üêç port_scanner.py
from argparse import ArgumentParser

parser = ArgumentParser(description="Scan ports on a host")

# Positional argument (required)
parser.add_argument("host", help="Target hostname")

# Optional argument with short form
parser.add_argument("-p", "--port", type=int, default=80,
                    help="Port to scan (default: 80)")

# Flag (boolean)
parser.add_argument("-v", "--verbose", action="store_true",
                    help="Verbose output")

args = parser.parse_args()
print(f"Scanning {args.host}:{args.port}")
if args.verbose:
    print("Verbose mode enabled")

üîß Subcommands

Like git commit, git push ‚Äî different modes with different arguments:

üêç network_tool.py
parser = ArgumentParser(description="Network tool")
subparsers = parser.add_subparsers(dest="mode", required=True)

# Server mode
server_parser = subparsers.add_parser("server")
server_parser.add_argument("--port", type=int, default=8080)

# Client mode
client_parser = subparsers.add_parser("client")
client_parser.add_argument("--host", required=True)
client_parser.add_argument("--port", type=int, default=8080)

args = parser.parse_args()
if args.mode == "server":
    start_server(args.port)
elif args.mode == "client":
    connect_to(args.host, args.port)

What does action="store_true" do?

AStores the string "true"
BSets True if flag present, False otherwise
CRequires a boolean value after the flag
DValidates input is true or false
Hint 1/3

Think about flags like -v or --verbose.

Hint 2/3

The flag itself is the signal, no value needed.

Hint 3/3

If --verbose is present ‚Üí True, if absent ‚Üí False.
