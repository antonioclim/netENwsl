‚ö†Ô∏è Common Error

üîÑ encode() and decode()

üêç encoding_basics.py
# String to bytes (before sending)
message = "Hello, Network!"
data = message.encode('utf-8')
print(data)  # b'Hello, Network!'

# Bytes to string (after receiving)
received = b'Response from server'
text = received.decode('utf-8')
print(text)  # "Response from server"

# In socket code:
sock.sendall(message.encode('utf-8'))
response = sock.recv(1024).decode('utf-8')

üí° Rule

send() needs bytes ‚Üí use encode() | recv() returns bytes ‚Üí use decode()

üìù List Comprehensions

Concise syntax for creating lists from iterations:

üêç list_comprehensions.py
# Traditional loop
ports = []
for p in range(1, 1001):
    if p % 100 == 0:
        ports.append(p)

# Same with list comprehension
ports = [p for p in range(1, 1001) if p % 100 == 0]
# [100, 200, 300, ..., 1000]

# Dict comprehension for HTTP headers
raw = "Host: example.com\r\nContent-Type: text/html"
headers = {
    k.strip(): v.strip()
    for line in raw.split('\r\n')
    for k, v in [line.split(':', 1)]
}

üåê IP Address Conversions

üêç ip_conversions.py
import socket

# String IP to bytes (for packet construction)
ip_str = "192.168.1.1"
ip_bytes = socket.inet_aton(ip_str)
print(ip_bytes)       # b'\xc0\xa8\x01\x01'
print(ip_bytes.hex()) # 'c0a80101'

# Bytes to string IP
ip_back = socket.inet_ntoa(ip_bytes)
print(ip_back)  # '192.168.1.1'

# Hex breakdown:
# c0 = 192, a8 = 168, 01 = 1, 01 = 1

‚ö†Ô∏è Critical Rule: Binary Data

When Forwarding Data (Proxy/Tunnel)

Do NOT decode and re-encode! Binary data will be corrupted.

üêç proxy_forward.py
# ‚ùå WRONG - corrupts binary data
data = client.recv(1024)
text = data.decode('utf-8')  # Fails on images!
server.send(text.encode('utf-8'))

# ‚úÖ CORRECT - forward bytes as-is
data = client.recv(1024)
server.sendall(data)  # Works for any data
