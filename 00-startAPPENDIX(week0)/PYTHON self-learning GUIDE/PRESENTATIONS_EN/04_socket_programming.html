üì¶ TCP vs UDP

üîµ TCP (SOCK_STREAM)

  • Connection-oriented (handshake required)
  • Guarantees delivery and order
  • Retransmits lost packets

Used for: HTTP, SSH, Email

üü† UDP (SOCK_DGRAM)

  • Connectionless (fire and forget)
  • No delivery guarantee
  • Minimal overhead, low latency

Used for: DNS, Gaming, VoIP

üíª TCP Server Code

üêç server_tcp.py
import socket

HOST = "127.0.0.1"
PORT = 9090

# Context manager: socket closes automatically!
with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as server:
    server.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
    server.bind((HOST, PORT))
    server.listen(5)
    print(f"üñ•Ô∏è Server listening on {HOST}:{PORT}")
    
    while True:
        conn, addr = server.accept()
        with conn:
            print(f"‚úÖ Connection from {addr}")
            data = conn.recv(1024)
            if data:
                conn.sendall(data.upper())

üêç client_tcp.py
import socket

HOST = "127.0.0.1"
PORT = 9090

with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as client:
    client.connect((HOST, PORT))
    print(f"üîó Connected to {HOST}:{PORT}")
    
    message = "Hello Server!"
    client.sendall(message.encode('utf-8'))
    print(f"üì§ Sent: {message}")
    
    response = client.recv(1024)
    print(f"üì• Response: {response.decode('utf-8')}")

üí° send() vs sendall()

send() may send less than requested. sendall() guarantees complete send.

üü† UDP ‚Äî Connectionless

UDP Server

with socket.socket(socket.AF_INET, socket.SOCK_DGRAM) as s:
    s.bind(("127.0.0.1", 9091))
    while True:
        data, addr = s.recvfrom(1024)
        s.sendto(b"ACK", addr)

UDP Client

with socket.socket(socket.AF_INET, socket.SOCK_DGRAM) as s:
    server = ("127.0.0.1", 9091)
    s.sendto(b"Hello UDP!", server)
    data, addr = s.recvfrom(1024)

üîë Key Differences

TCP: SOCK_STREAM, connect(), send()/recv()

UDP: SOCK_DGRAM, sendto(addr)/recvfrom()

üîí Context Managers (with)

Context managers guarantee resources are released, even if exceptions occur.

‚ùå Without with (risky)

sock = socket.socket(...)
sock.connect((host, port))
sock.send(data)  # Exception here?
sock.close()     # Never called!

‚úÖ With with (safe)

with socket.socket(...) as sock:
    sock.connect((host, port))
    sock.send(data)
# Auto-closed here, guaranteed!

üìù Quiz

üß†

Which socket type is for TCP?

Asocket.SOCK_DGRAM
Bsocket.SOCK_STREAM
Csocket.SOCK_RAW
Dsocket.SOCK_TCP
Hint 1/3

TCP = "stream" of continuous, ordered data.

Hint 2/3

UDP = "datagrams". There's no SOCK_TCP constant.

Hint 3/3

SOCK_STREAM for TCP because it provides a byte stream.

üß†

Correct order for TCP server?

Asocket ‚Üí listen ‚Üí bind ‚Üí accept
Bsocket ‚Üí bind ‚Üí listen ‚Üí accept
Csocket ‚Üí connect ‚Üí listen ‚Üí accept
Dsocket ‚Üí accept ‚Üí bind ‚Üí listen
Hint 1/3

You must bind to an address before you can listen on it.

Hint 2/3

listen() marks socket as passive, accept() waits for connections.

Hint 3/3

connect() is for clients, not servers!

‚ö†Ô∏è Common Errors

Address already in use

Port is still in TIME_WAIT state.

Fix: setsockopt(SOL_SOCKET, SO_REUSEADDR, 1)

Connection refused

Server not running or wrong port.

Fix: Check server is listening on correct port.

Timeout

Server not responding.

Fix: Set timeout: sock.settimeout(5.0)

TypeError: bytes required

Sending str instead of bytes.

Fix: Use .encode('utf-8')
